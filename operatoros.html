OperatorOS

A CLI for managing personal operating-system workspaces — your scripts, notes, secrets, and tools, organized into a typed, composable, version-controlled structure you can back up, share, and extend.

The problem

You are one operator. You run a workspace that is supposed to be yours — files, scripts, notes, tools, agents, scheduled jobs, integrations. In practice it has fragmented into:

  • A ~/projects/ tree that has drifted since 2023
  • Three different note systems, none of which the others can read
  • AI agents that each invented their own conventions
  • A bash history that is the actual source of truth for 40% of your workflows
  • Secrets scattered across .env, gpg, and that one Notion page

None of this is a single system. None of it is replaceable. None of it composes.

Six principles

01 · Single Authority

Every concept has one canonical owner file. No dual sources. If two files disagree, one is wrong.

02 · Everything Replaceable

Modules, presets, even Core itself — all replaceable. Framework first, implementation last.

03 · Typed Substrate

Every artifact validated against a JSON Schema at the boundary. Invalid state cannot exist.

04 · AI-native

Core ships ≥1 of: agent-loop primitive, structured-output validation, prompt-template versioning, LLM-provider abstraction.

05 · Composable Modules

A module is one composable extension. It declares a contract (module.yaml), runs in isolation, exposes one CLI surface.

06 · Profile-portable

Your whole workspace exports to a single bundle. Move to a new machine in 10 minutes. Publish, fork, or archive at will.

Quickstart (v0.5.1-alpha)

# Install the single-file binary (Node 20+ required)
curl -fsSL https://raw.githubusercontent.com/taras-polishchuk/operatoros-framework/main/scripts/install.sh | sh

# Initialize a workspace
mkdir ~/my-workspace && cd ~/my-workspace
operatoros init

# Apply the default preset — installs two example modules (journal + timer)
operatoros apply

# Try them
operatoros run journal add "shipped v0.5.1-alpha"
operatoros run timer start "deep work" 50
operatoros run timer start "defaulted task"     # falls back to $DEFAULT_MINUTES from settings
operatoros run timer list

# Export the workspace as a portable bundle
operatoros export --bundle tar.gz
# → ~/my-workspace-2026-07-02T23-50-00.tar.gz (secrets excluded)

Commands

apply [preset]Install all modules declared in a preset
run <m> <cmd>Execute a module's declared command

versionPrint version + git info

What it is NOT for

OperatorOS is not for teams — pick a team-OS if you have one. It is not a SaaS — there's no hosted OperatorOS, it runs locally. It is not a graph database, static site generator, chat framework, or agent runtime — those are modules you install on top of OperatorOS, or you write them yourself.

OperatorOS is for one operator: the human (and their AI agents) who want their workspace to be a system, not a folder.
