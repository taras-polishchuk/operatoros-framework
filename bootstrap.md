## 5. Onboarding

**If `IDENTITY.md` lacks `onboarding_complete: true`:**

1. Run the onboarding interview. See `methodology/05-onboarding-interview.md`.
2. The interview has 5 questions. Don't add more.
3. Write the answers to `IDENTITY.md` as a structured addendum.
4. Set `onboarding_complete: true` and `last_updated: <today>`.
5. Future cold-starts skip the interview.

**If `IDENTITY.md` has `onboarding_complete: true`:**

1. Check the `last_updated` field.
2. If > 6 months old, offer to re-interview. Don't run it unilaterally.
3. If recent, proceed with the recorded parameters.

## 6. Mission lifecycle

When you start work that produces more than one file or takes more than 30 minutes:

1. Create `.project-state/<mission-slug>/` with the 8 standard artifacts.
2. Use `mission-execution-model` for guidance on mission structure.
3. Update `progress.md` after each meaningful checkpoint, not every tool call.
4. On completion, write `final-report.md` and move mission to `done` state.

## 7. When in doubt

- **Don't invent.** Read the canonical source. If it doesn't exist, ask.
- **Don't duplicate.** Reference by path. Don't copy content.
- **Don't overshoot.** Do the minimum needed for the task.
- **Don't undershoot.** If the task is non-trivial, use the mission lifecycle.

**Default uncertainty behavior:** `default-and-log`. Make a reasonable choice, write it to `decisions.md` in the mission directory, continue. Ask only if blocked by credentials, legal, or explicit user request.
